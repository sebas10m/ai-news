What is an Agent?

An agent is essentially the core LLM, the intelligence, wrapped with memory, tools, and the ability to collaborate with other agents.

When to Use Agents

When building applications with LLMs, we recommend finding the simplest solution possible and only increasing complexity when needed.

Agent Frameworks

Agentic frameworks are powerful, and they're getting more powerful by the day.

Agent Patterns

  • Prompt Chaining: breaking down a complex task into a sequence of steps
  • Routing: sending a prompt to multiple agents to determine the best solution
  • Parallelization: having multiple agents work together to decrease latency
  • Orchestrator Workers: a central LLM breaking down tasks and delegating them to worker LLMs
  • Enumerator Optimizer: one LLM generating a response and another providing evaluation and feedback in a loop

Agent Implementations

Agents can be used for open-ended problems where it's difficult or impossible to predict the number of steps and where you can't hard code a fixed path.

Conclusion

The key to success is measuring performance and iterating on implementations.
