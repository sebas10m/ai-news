Nixie Search: An Open Source Search Engine

Nixie Search is a search engine built on Apache Lucene, with a novel approach to indexing and querying. It uses a block storage index in S3, allowing for highly available and scalable search functionality.

Key Features

  • Native language support
  • Facets and filters
  • Query completion and suggestions
  • Semantic search using embedding models
  • Query generation model for fine-tuning
  • Robust and well-tested codebase

Benefits

  • Improved search results and relevance
  • High availability and scalability
  • Easy to use and maintain
  • Fine-tuning and customization options

Technical Details

Nixie Search is built using a combination of Apache Lucene, Apache Kafka, and Docker. It uses a block storage index in S3 for efficient querying and indexing.

Demo and Evaluation

Roman has demonstrated the search functionality of Nixie Search using a movie lens dataset. The demo showcases the search engine's ability to perform facets, aggregations, filtering, and autocomplete.

Comparison to MetaRank

Nixie Search is a more comprehensive search engine compared to MetaRank, which is a learned ranker for secondary ranking. While MetaRank focuses on solving the problem of broken relevancy, Nixie Search provides a proper search engine with fine-tuning options.

Advice for Implementers

Roman advises developers to start with the basics of information retrieval, focusing on BM25 and lexical retrieval, which are strong baselines in production. He also emphasizes the importance of proper search engine design and not relying on hype or overambitious technologies.

Conclusion

Nixie Search is an open-source search engine built on top of Apache Lucene, with a novel approach to indexing and querying. Its key features, benefits, and technical details make it an attractive choice for developers looking to build a robust and scalable search functionality.
