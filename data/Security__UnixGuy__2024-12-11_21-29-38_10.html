Understanding Wi-Fi Attacks and Cracking WPA and WPA2

Wi-Fi networks are notoriously known to be vulnerable to cyber attacks. In fact, one of the most common ways to break into an organization is to drive really close to that organization and try to get that initial access by compromising their Wi-Fi.

Types of Wi-Fi Attacks

  • The evil twin attack, where a hacker creates a fake access point that usually has a similar name to the real access point.
  • The rogue access point, where a hacker is really close to the network or perhaps even inside the network and they set up a rogue Wi-Fi that looks exactly like the real one.
  • The WPS attack, where the WPS setup was created because some people didn't like to type a long password or instead they created an eight digit pin to make connection easier.
  • The WPA or WPA2 attack, where WPA or WPA2 uses strong password combinations and the WPA2 is the strongest encryption algorithm.

Cracking WPA and WPA2

WPA or WPA2 uses a strong password combination and the four-way handshake, where the server sends a challenge to the client and then the client encrypts that challenge and sends it back to the server. This is vulnerable to a type of man-in-the-middle attack where the attacker can sniff that challenge and then perform brute force attacks against that encrypted challenge text to try and determine that PSK or password.

Capturing the Encrypted Challenge

We need to capture the encrypted challenge to be able to brute force it. We can do this by turning our wireless network interface into monitoring mode and sniffing traffic from the network. We want to disconnect the client from the server to force it to re-establish the four-way handshake again and capture that challenge.

Executing the Commands

We need to run two terminals: one to sniff the packets on the network and the other to execute commands such as authenticating the clients and forcing it to reconnect. The commands we need to run are: iwconfig to show our wireless interface, iwscan to scan our network for wireless access points, IP set dev wlan2 down to set the link down, iwconfig wlan2 set type monitor to change the mode to monitoring, aerodump-ng to sniff packets and capture the encrypted challenge, aireplay-ng to send authentication packets to disconnect the client and force it to re-establish the handshake, and finally aircrack-ng to brute force the captured challenge and obtain the PSK.
