A Couple of Techniques to Test for Broken Access Control

Difference Between Authentication and Authorization

Authentication and authorization are two distinct concepts. Authentication is about verifying one's identity, while authorization is about what actions one is allowed to perform. In the context of a hotel, checking into a room is an authentication process (verifying identity), while accessing the room with a key card is an authorization process (determining allowed actions).

Important Lab Techniques

When testing API endpoints, it's essential to try different HTTP verbs (GET, POST, PUT, DELETE, etc.) to discover potential vulnerabilities. Additionally, thoroughly testing multi-step sequences, such as registration or checkout processes, is crucial since they can introduce issues and vulnerabilities.

When discovering vulnerable endpoints, one can modify them to introduce bugs and unauthorized actions. Using JWT tokens, it's possible to test their validity, one-time use, and impact on different users.

  • Check if a hidden PUT endpoint can be used without authentication.
  • Verify if a reset token is one-time use or can be reused.
  • Determine if the application checks the user ID or uses the JWT token for authorization.
  • Test for vulnerabilities in token generation and validation.
  • Explore edge cases, such as using null or zero as a token value.

Understanding how to test access control in an application is crucial for discovering vulnerabilities and ensuring the security of user data.
