DLT REST API Source Presentation

Willi Müller, a data engineering freelancer and partner of Untitled Data Company, demonstrated the DLT REST API source during Open Source Spotlight. DLT is a library used to quickly build data pipelines and load data from unexpected places.

DLT REST API Overview

The DLT REST API source allows users to declare REST API pipelines using Python dictionaries, making it easy to write and maintain. It supports features like authentication, pagination, and processor steps, making it a powerful tool for loading data from REST APIs.

Configuration with Dictionaries

The configuration is done using Python dictionaries, which can be executed as code. This makes it easy to debug and maintain.

Example Configuration

import dlt

# Declare a REST API pipeline
dlt.model('pokemon', source="pokemon-api")

# Load data from the pipeline
# In this example, we load the 'berries' table from the 'pokemon-api' source
data = dlt.load('pokemon', 'berries')

Executing the Pipeline

After configuring the pipeline, it can be executed using the DLT REST API client.

Executing the Pipeline with Debugging

# Execute the pipeline with debugging
dlt.run('pokemon', debug=True)

Overriding the REST API

The DLT REST API source allows users to override certain configurations, such as authentication and pagination.

Overriding Authentication

# Override authentication using a custom callable
auth = lambda x: {'headers': {'Authorization': 'Bearer YOUR_TOKEN'}}

Incremental Loading

The DLT REST API source supports incremental loading, which allows users to load new data only when necessary.

Example Incremental Loading

# Enable incremental loading for the 'issues' table
dlt.model('issues', source='github-api', incremental=True, update_by='updated_at', filter_by='assignee = efficacy')

Conclusion

The DLT REST API source is a powerful tool for loading data from REST APIs using Python dictionaries. Its configuration-based approach makes it easy to write and maintain, and it supports features like authentication, pagination, and incremental loading.
